# Code of Conduct

This project should be a focused, respectful place to improve Kusabi /
wasurezu as an MCP memory and recovery tool.

## Expected Behavior

- Be respectful and direct.
- Assume good intent, but keep technical claims evidence-based.
- Review code, designs, and evidence rather than people.
- Keep issue and PR discussions on the scoped change.
- Avoid posting private transcripts, secrets, database dumps, recovery packs, or
  customer/operator data.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing another person's private information.
- Posting secrets, credentials, private recovery artifacts, or sensitive local
  paths without authorization.
- Repeatedly derailing issues or PRs after maintainers have clarified scope.
- Misrepresenting review, audit, legal, compliance, release, or publish
  approval.

## Reporting

For ordinary conduct concerns, contact the repository maintainers through a
private channel or open a narrowly scoped issue if no private information is
involved.

For security-sensitive reports, follow [SECURITY.md](SECURITY.md). Do not open a
public issue containing secrets, private transcripts, database contents, or
recovery artifacts.

## Enforcement

Maintainers may remove comments, close issues, block participation, or take
other repository-level moderation actions when behavior disrupts the project or
creates safety risk.

This Code of Conduct is community guidance for this repository. It does not
create legal, compliance, trademark, public release, or publish authorization.
